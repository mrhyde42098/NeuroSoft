## 🟠 ALTOS

### A1 · Atajos globales no chequean `contentEditable` ni Modales
- **Archivo:** `neurosoft-frontend/src/App.jsx:230`
- **Categoría:** atajos de teclado en inputs
- **Código actual:**
  ```js
  useEffect(()=>{const handler=e=>{if(!e.altKey)return;
    const tag=(e.target.tagName||"").toLowerCase();
    if(tag==="input"||tag==="textarea"||tag==="select")return;
    const sc=SHORTCUTS.find(s=>s.key.toLowerCase()===e.key.toLowerCase());
    if(sc){e.preventDefault();...}
  ```
- **Problema:** chequea `tag` pero no `e.target.isContentEditable`. Si el usuario está en un campo `contentEditable` (el editor de informe IA usa esto en algunos lugares) y pulsa `Alt+1`, navega fuera y pierde lo escrito.
- **Impacto clínico:** el clínico pierde la observación a medio escribir.
- **Sugerencia:** agregar `if (e.target?.isContentEditable) return;` antes del find.

### A2 · GoNoGo/CPT escuchan `Space` sin filtrar focus en inputs
- **Archivos:** `neurosoft-frontend/src/app/rehab/GoNoGoActivity.jsx:90`, `CPTActivity.jsx:86`
- **Categoría:** atajos de teclado en inputs
- **Código actual:**
  ```js
  const h = (e) => { if (e.code === "Space") respond(); };
  window.addEventListener("keydown", h);
  ```
- **Problema:** durante una actividad, si el clínico abre un modal con input de texto y pulsa space, dispara `respond()` y contamina la métrica clínica de la actividad.
- **Impacto clínico:** falso positivo en aciertos / omisiones.
- **Sugerencia:** añadir guard `const tag=(e.target?.tagName||"").toLowerCase(); if(tag==="input"||tag==="textarea") return;` antes del `e.code === "Space"`.

### A3 · `DashboardPage` — `adheres.indexOf(a)` frágil tras refactor
- **Archivo:** `neurosoft-frontend/src/app/dashboard/DashboardPage.jsx:48`
- **Categoría:** asunciones sobre orden de listas
- **Código actual:**
  ```js
  const conPlan = adheres.filter(a => a && a.has_plan);
  ...
  .map((a) => ({ ...a, paciente: pacs[adheres.indexOf(a)]?.nombre_completo || "—" }))
  ```
- **Problema:** funciona porque `Promise.all` preserva orden y `adheres` comparte referencias con `conPlan`. Pero si un día se introduce un `.map(x=>({...x}))` intermedio que clone los objetos, `indexOf(a)` devolverá `-1` y `pacs[-1]` → `undefined` → "—" silenciosamente para todos. Ya no funciona y nadie se entera.
- **Impacto clínico:** Dashboard mostraría "—" como nombre en lugar de los pacientes con peor adherencia.
- **Sugerencia:** indexar por id explícito: `const byId = Object.fromEntries(pacs.map(p=>[p.id, p])); ... byId[a.paciente_id]?.nombre_completo`.

### A4 · `report_pro` — `except Exception` silencioso en charts.py
- **Archivo:** `neurosoft-backend/app/infrastructure/report_pro/charts.py:341, 422`
- **Categoría:** errores tragados sin loggear
- **Código actual:**
  ```py
  try:
      ... # dibujar radar / discrepancias
  except Exception:
      pass  # ← sin log
  ```
- **Problema:** si falla el dibujo de un gráfico clínico (por datos faltantes / dimensiones inconsistentes), el PDF se genera sin él pero nadie ve el error. El clínico recibe un informe incompleto sin advertencia.
- **Impacto clínico:** informe entregado sin radar de dominios o discrepancias visibles.
- **Sugerencia:** `logger.warning("Fallo al dibujar gráfico %s: %s", chart_name, exc, exc_info=True)`.

### A5 · `auth_service` — fallback a `"neurosoft2025"` en modo dev no-frozen
- **Archivo:** `neurosoft-backend/app/infrastructure/auth/auth_service.py:327`
- **Categoría:** seguridad por defecto débil
- **Código actual:**
  ```py
  if getattr(_sys, "frozen", False):
      password = secrets.token_urlsafe(40)
  else:
      password = "neurosoft2025"  # ← fallback dev
  ```
- **Problema:** en cualquier instalación dev (clones de repo) la contraseña admin queda `neurosoft2025`. Si alguien expone el puerto 8000 por accidente, root access trivial. La defensa actual (env=development) es buena pero el secreto debería ser aleatorio.
- **Impacto clínico:** acceso administrativo no autorizado a HC.
- **Sugerencia:** generar siempre `secrets.token_urlsafe(40)` y persistirla a `<APPDATA>/NeuroSoft/admin_password_dev.txt` con `chmod 600`. Loggear "Admin password generada — ver /path/al/archivo" UNA vez.

## 🟡 MEDIOS

### M1 · Migración Alembic faltante para `therapy_tasks`
- **Archivo:** `neurosoft-backend/alembic/versions/` (no existe migración `005_*`)
- **Categoría:** consistencia de schema
- **Problema:** la tabla nueva `therapy_tasks` se crea por `Base.metadata.create_all()` al arranque (que es idempotente). Funciona, pero queda fuera del control de versiones de Alembic, así que `alembic upgrade head` no la incluye y `alembic current` no la conoce.
- **Impacto:** en deployments con Alembic estricto (ej. servidor central futuro), la tabla no se crearía si `create_all()` no se llama. Para Mayra (SQLite local) no hay impacto.
- **Sugerencia:** generar `alembic/versions/005_therapy_tasks.py` con `op.create_table("therapy_tasks", ...)` espejo del ORM.

### M2 · `ReactivePanel` — `parseInt(scores[k])||0` sin radix en scoring clínico
- **Archivo:** `neurosoft-frontend/src/app/evaluation/ReactivePanel.jsx:26, 28, 29, 30, 31, 32, 33, 73, 85, 93, 95, 120, 135, 191`
- **Categoría:** validación numérica
- **Código actual:**
  ```js
  total+=(parseInt(next[`i${it.n}`])||0)
  ```
- **Problema:** `parseInt("08")` en ES5 daba 0 (octal). En navegadores modernos da 8, pero pywebview usa EdgeChromium que es moderno → OK en práctica. Cosmético en código clínico.
- **Impacto clínico:** ninguno con browsers modernos, pero buena práctica defensiva en scoring.
- **Sugerencia:** `parseInt(x, 10) || 0` en los 14 sitios. Tarea trivial con replace-all bien acotado.

### M3 · `window.confirm()` legado en EvalResultsPage
- **Archivo:** `neurosoft-frontend/src/app/evaluation/EvalResultsPage.jsx:170`
- **Categoría:** UX inconsistente
- **Código actual:**
  ```js
  const ok=window.confirm(`Se detectaron ${pedWarningCount} advertencia(s)...`);
  ```
- **Problema:** el comentario en línea 47 dice "usamos toast en lugar de alert() — antes había 9 alert() bloqueantes" pero este `window.confirm()` sobrevivió. Es bloqueante y feo en dark mode.
- **Sugerencia:** crear un modal de confirmación reutilizable basado en `ToastProvider` o un componente `ConfirmModal` que se invoque con promesa.

### M4 · `alert()` superviviente para "Texto copiado"
- **Archivo:** `neurosoft-frontend/src/app/evaluation/EvalResultsPage.jsx:262`
- **Categoría:** UX inconsistente
- **Código:**
  ```js
  navigator.clipboard?.writeText(wiscReportText);
  alert("Texto copiado al portapapeles")
  ```
- **Sugerencia:** `toast.success("Texto copiado")` (ya tienes el hook arriba en la función).

### M5 · `localStorage` sin try/catch en ~14 archivos
- **Archivos:** App.jsx, contexts.jsx, NotificationsWidget, etc. (49 ocurrencias de getItem/setItem, solo 4 con try/catch)
- **Categoría:** robustez
- **Problema:** modo privado del navegador, política de storage del sistema, o cuota llena → `setItem` lanza `QuotaExceededError`. Sin try/catch, rompe el flujo. Riesgo bajo en pywebview, alto si se sirve la SPA pública vía /shared/.
- **Sugerencia:** helper `safeLS = { get(k){try{return localStorage.getItem(k)}catch{return null}}, set(k,v){try{localStorage.setItem(k,v)}catch{}} }` y usar en toda la app.

### M6 · CORS default `"*"` en config
- **Archivo:** `neurosoft-backend/app/core/config.py:301`
- **Categoría:** seguridad
- **Código:**
  ```py
  cors_origins = ["*"]
  ```
- **Problema:** default permisivo. main.py degrada a `allow_credentials=False` cuando hay `"*"` (correcto), pero un valor por defecto más restrictivo (`["http://localhost:5173", "http://127.0.0.1:8765"]`) sería más seguro.
- **Sugerencia:** restringir a localhost para dev, exigir override explícito en prod.

### M7 · `bibliotecaRecursos.js` — strings largos hardcoded sin i18n
- **Archivo:** `neurosoft-frontend/src/data/bibliotecaRecursos.js`
- **Categoría:** mantenibilidad
- **Problema:** 45 fichas bibliográficas como strings en el bundle (33 KB). Si en el futuro Mayra agrega más, el bundle crece sin control. Y si se quiere editar sin recompilar → imposible.
- **Sugerencia:** mover a `data/biblioteca.json` y cargar con `import biblio from "./biblioteca.json"`. Mejor aún: servir desde backend `/api/v1/biblioteca` para edición sin redeploy.

### M8 · Service Worker `controllerchange` puede causar bucle de recarga
- **Archivo:** `neurosoft-frontend/index.html:130-142`
- **Categoría:** PWA / SW
- **Código:**
  ```js
  navigator.serviceWorker.addEventListener('controllerchange', () => {
    if (reloaded) return;
    reloaded = true;
    window.location.reload();
  });
  ```
- **Problema:** con el listener combinado con `skipWaiting` automático, es posible que el primer load de cada arranque haga un reload extra (cuando el SW se promueve de `installing` a `controlling`). El guard `reloaded` evita bucle infinito pero hay un re-render visible.
- **Sugerencia:** mostrar un mini-toast "Actualizando NeuroSoft a la última versión…" antes del reload, o disparar el reload solo si `window.NEUROSOFT_INITIAL_LOAD === false`.

### M9 · `parseInt(next[k])||0` — el `||0` enmascara `NaN`
- **Archivo:** `neurosoft-frontend/src/app/evaluation/ReactivePanel.jsx:26-33`
- **Categoría:** validación
- **Problema:** si el clínico escribe `"abc"` por error en un campo PD, `parseInt("abc")||0` da 0 silenciosamente y el scoring usa PD=0 (resultado clínico erróneo, no flag).
- **Sugerencia:** validar y mostrar warning visible si la conversión falla.
