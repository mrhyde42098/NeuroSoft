## Hallazgos detallados

### [CRÍTICO] IDOR — Historia clínica y PDF sin verificación de ownership

- **Archivo:línea:** `neurosoft-backend/app/presentation/api/v1/clinical_history.py:89-134`
- **Descripción:** `get_clinical_history`, `generate_hc_pdf` e interpretación usan solo `Depends(get_session)`. No hay `CurrentUser` ni `get_patient_for_user` (a diferencia de `patients.py`).
- **Impacto:** Usuario autenticado con UUID de otro paciente puede leer HC completa y descargar PDF con PHI (Ley 1581 / Res. 1995).
- **Reproducción:** Login como profesional A → `GET /api/v1/clinical-history/{patient_id_de_B}` con token válido.
- **Fix sugerido:** Mismo patrón que pacientes:

```python
def get_clinical_history(
 patient_id: str,
 db: Session = Depends(get_session),
 user: User = Depends(get_current_user),
):
 get_patient_for_user(patient_id, db, user)
 return GetClinicalHistoryUseCase(db).by_patient(patient_id)
```

---

### [CRÍTICO] Pérdida de advertencia etaria cuando también hay `out_of_baremo`

- **Archivo:línea:** `neurosoft-backend/app/domain/clinical_engine/engine.py:394-415`
- **Descripción:** Una sola variable `warn` se asigna para rango etario y luego se **sobrescribe** si `out_of_baremo`. Solo una advertencia llega al informe.
- **Impacto:** Clínico no ve alerta de baremo inapropiado por edad (p. ej. subtest `Ni*` en adulto mayor) si además el PD está fuera de tabla.
- **Reproducción:** Paciente 65 años + test `NiWiscSem` + PD fuera de baremo.
- **Fix sugerido:** Lista acumulativa:

```python
advertencias: list[str] = []
if rango is not None and not (min_edad <= ctx.age.years <= max_edad):
 advertencias.append(f"'{prueba.nombre}' está baremado para {pob_esperada} ...")
if output.metadata.get("out_of_baremo"):
 advertencias.append(f"'{prueba.nombre}' (PD={pd}): fuera del rango del baremo ...")
# asignar advertencias al ResultadoPrueba (extend field o join)
```
