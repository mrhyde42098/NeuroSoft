## 🔴 CRÍTICOS (2)

### C1 · Inconsistencia clínica en `ZScoreMultipleStrategy` con sigma=0
- **Archivo:** `neurosoft-backend/app/domain/clinical_engine/strategies.py:320`
- **Categoría:** División por cero / falso positivo clínico
- **Código actual:**
  ```python
  z = round((pd - mu) / sigma, 2) if sigma != 0 else 0.0
  ```
- **Problema:** Mientras que `ZScoreStrategy` (línea 291) marca `sigma=0` como `sin_norma` (motivado por el fix §H9-fix), `ZScoreMultipleStrategy` retorna **silenciosamente 0.0** (= "rendimiento promedio"). Esto es un **FALSO POSITIVO clínico**: un baremo corrupto aparecería como normal.
- **Impacto clínico:** Un paciente con baremo corrupto en CARAS-R (única prueba que usa esta strategy) recibiría Z=0.0 (Promedio). Puede sesgar un informe completo.
- **Sugerencia:** Replicar el patrón de `ZScoreStrategy` líneas 291-298 — retornar `ScoringOutput` con `puntaje_escalar=None` y `metadata={"sin_norma": True, "motivo": "sigma=0 en baremo"}`.

### C2 · `scores.length` sin guard en `TowerOfLondonActivity`
- **Archivo:** `neurosoft-frontend/src/app/rehab/TowerOfLondonActivity.jsx:171`
- **Categoría:** División por cero (rehab clínica)
- **Código actual:**
  ```js
  if (done) {
    const avg = scores.reduce((s, r) => s + r.efficiency, 0) / scores.length;
  ```
- **Problema:** Si el usuario termina la actividad con 0 trials completados (cancelación tardía, bug del state machine), `scores` es `[]` y la división da `NaN`. Se muestra en el resultado al clínico.
- **Impacto clínico:** Resultado rehab aparece como `NaN` en el panel del paciente.
- **Sugerencia:** `const avg = scores.length ? scores.reduce(...) / scores.length : 0;`

## 🟠 ALTOS (5)

### A1 · `progressPct` sin guard en `EvalApplyPage`
- **Archivo:** `neurosoft-frontend/src/app/evaluation/EvalApplyPage.jsx:266`
- **Categoría:** División por cero
- **Código actual:**
  ```jsx
  <div style={{width:`${(done.length/tests.length)*100}%`}}/>
  ```
- **Problema:** Si `tests` está vacío (caso raro: test sin items, race condition de carga), la barra de progreso da `Infinity%` o `NaN%`.
- **Sugerencia:** `\`${tests.length ? (done.length/tests.length)*100 : 0}%\``

### A2 · 7 `localStorage.getItem` sin try/catch en rutas de producción
- **Archivos:**
  - `EvalApplyPage.jsx:32` — `localStorage.getItem("ns_sel_patient")`
  - `EvalApplyPage.jsx:48` — `localStorage.getItem(...)` dentro de reduce
  - `HistorialPage.jsx:24` — `localStorage.getItem("ns_sel_patient")`
  - `AgendaPage.jsx:21` — `localStorage.getItem("ns_agenda_view")`
  - `NotificationsWidget.jsx:57,66` — `localStorage.getItem(LAST_SEEN_KEY)`
  - `App.jsx:107` — `localStorage.getItem(KEY)`
- **Categoría:** Crash por modo privado / quota
- **Problema:** Si el usuario está en modo privado (Safari, Brave), `localStorage.getItem` lanza `SecurityError`. La app **no monta**.
- **Impacto:** Pantalla en blanco al primer login en navegador privado. El usuario no puede usar la app.
- **Sugerencia:** Existe `src/utils/safeLS.js` con helper `getLS(key, default)`. Usar en lugar de localStorage directo en estos 7 sitios.

### A3 · `window.prompt` nativo en producción
- **Archivo:** `neurosoft-frontend/src/app/therapy/SesionSOAPForm.jsx:363`
- **Categoría:** UX inconsistente
- **Código actual:**
  ```js
  window.prompt("Copia este link:", url);
  ```
- **Problema:** El modal nativo del navegador rompe la UI. CLAUDE.md frontend dice: usar `ToastProvider` o componente modal.
- **Sugerencia:** Reemplazar con `toast.info("Link copiado al portapapeles")` + `navigator.clipboard.writeText(url)`.

### A4 · `window.confirm` nativo en producción (×2)
- **Archivos:**
  - `neurosoft-frontend/src/app/aprender/EstudiarPage.jsx:82` — confirmar reinicio de flashcards
  - `neurosoft-frontend/src/app/patients/CompanionsSection.jsx:96` — confirmar eliminar acompañante
- **Categoría:** UX inconsistente / accesibilidad
- **Problema:** Mismo problema que A3. El contexto ya provee `ConfirmProvider` + `useConfirm()` (importado en otros archivos).
- **Sugerencia:** Reemplazar con `const confirm = useConfirm(); if (await confirm("¿...?")) ...`

### A5 · `baremoLoader` sin manejo de error en `engine.py` si BD_NEURO_MAESTRA.json no existe
- **Archivo:** `neurosoft-backend/app/main.py:97-118`
- **Categoría:** Defensive coding
- **Problema:** Si la app arranca sin el JSON (build roto, file corruption), solo se loggea `warning` y se continúa. El endpoint `/score` crasheará con `KeyError` no controlado. (Verificado que el `try/except` en `engine.py:347` SÍ lo captura, pero el stacktrace aparece en cada request — UX horrible).
- **Sugerencia:** Al detectar BD_NEURO_MAESTRA.json ausente al startup, retornar **503 Service Unavailable** en `/score` con mensaje claro: "BD de baremos no cargada — contacte soporte".

## 🟡 MEDIOS (6)

### M1 · 9 `api.get().catch(() => {})` fallan silenciosamente
- **Archivos:** 9 archivos frontend
- **Categoría:** Logging silencioso
- **Problema:** Si el token JWT expira mientras el usuario navega, ve UI vacía sin explicación.
- **Sugerencia:** Cambiar `api.get().then(setX).catch(() => {})` → `api.get().then(setX).catch((e) => toast.error("Error cargando X"))`. Aplica a:
  - `HistorialPage.jsx:36,37,73`
  - `DashboardPage.jsx:28`
  - `RehabPage.jsx:94,671`
  - `TherapyPage.jsx:64`
  - `ScreeningPage.jsx:107`
  - `ComparePage.jsx:46`
  - `PatientsPage.jsx:38` (RegisterPage)
  - `InformesPage.jsx:52`

### M2 · 119 `key={i}` con index en listas
- **Archivos:** 30+
- **Categoría:** Code smell, riesgo de re-render
- **Problema:** Listas dinámicas con `key={i}` rompen reconciliación cuando se insertan/eliminan elementos del medio.
- **Impacto:** Performance sub-óptima, pero funcionalmente correcto en la mayoría de casos.
- **Sugerencia:** Usar IDs únicos cuando existan (`item.id`, `r.test_id`). Para arrays de strings, `key={item}`.

### M3 · `EvalResultsPage.jsx:177` — `try{...}catch{}` con side effects sin log
- **Archivo:** `neurosoft-frontend/src/app/evaluation/EvalResultsPage.jsx:177`
- **Categoría:** Logging silencioso
- **Problema:**
  ```js
  try{localStorage.removeItem(obsDraftKey())}catch{}
  ```
- **Sugerencia:** Usar `safeLS.remove(obsDraftKey())` desde `src/utils/safeLS.js`.

### M4 · `AuditoriaTab.jsx`, `AdminKpisTab.jsx`, `EstimulosTab.jsx` — `useEffect load() con [] deps`
- **Archivos:**
  - `neurosoft-frontend/src/app/config/AdminKpisTab.jsx:114`
  - `neurosoft-frontend/src/app/config/AuditoriaTab.jsx`
  - `neurosoft-frontend/src/app/config/EstimulosTab.jsx`
- **Categoría:** ESLint react-hooks/exhaustive-deps
- **Problema:** `useEffect(() => { load() }, [])` — `load` se redefine en cada render (closure stale) pero no está en deps.
- **Sugerencia:** Agregar `// eslint-disable-next-line react-hooks/exhaustive-deps` con comentario, O usar `useCallback` para `load`.
